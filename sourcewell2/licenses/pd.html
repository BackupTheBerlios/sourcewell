Public Domain

If you release your package to the public domain, anyone can do whatever they like with it.

There's no license, which avoids the need for disclaimers etc. However, you should declare unambiguously that you have released it to the public domain.

Here's how you can do it, in a way that incorporates "gentle persuasion" to encourage users of the package to propagate their enhancements back into the Eiffel community:

  • Put the following text into a file public.txt within your distribution:
 
This package is released to the public domain.

If you modify it, you are strongly encouraged to publicly release the modified version.
  • Near the top of each class file (or other item within the package), and in any announcements about the package, and in any application "splash screen" or "about box", include the following line:
 
Released to the public domain by <your-name>, see file "public.txt"
